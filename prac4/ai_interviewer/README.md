# Варіант 6. AI Interviewer для технічних спеціалістів

### [Інтерфейс (main.py)](src/interviewer/main.py) | [Логіка Gemini (core.py)](src/interviewer/core.py) | [Схеми (schemas.py)](src/interviewer/schemas.py)

#### Опис:
Проєкт (КП4) реалізує симулятор технічної співбесіди на базі великої мовної моделі від Google (`gemini-3.1-flash-lite`). 
Застосунок виступає у ролі Senior-розробника, який проводить інтерв'ю з кандидатом на обрану посаду (Junior/Middle/Senior). 
Модель послідовно ставить технічні питання, аналізує відповіді користувача у реальному часі, підтримує повний контекст бесіди, а по завершенню генерує розгорнутий фідбек (Scorecard) із оцінками та рекомендаціями. 
Веб-інтерфейс реалізовано за допомогою фреймворку **Streamlit**. 
Управління залежностями здійснюється через менеджер **uv**.

## 1. Архітектура діалогу (ChatSession)
Оскільки співбесіда вимагає запам'ятовування попередніх питань та відповідей, у проєкті замість одиничних запитів використовується об'єкт `ChatSession`. 
Метод `model.start_chat(history=[])` створює сесію, яка автоматично зберігає історію повідомлень. Це дозволяє моделі аналізувати повноту відповіді та ставити уточнюючі запитання без необхідності вручну передавати весь попередній текст.  
<sub>(+ економія і так невеликої кількості токенів)</sub>


```python
# Ініціалізація сесії зі збереженням стану (core.py)
model = genai.GenerativeModel(
    model_name='gemini-3.1-flash-lite',
    system_instruction=current_instruction,
    generation_config=genai.types.GenerationConfig(temperature=0.4, top_p=0.8)
)
return model.start_chat(history=[])

```

## 2. Рольова модель (System Instructions)

Для налаштування поведінки ШІ використано механізм `system_instruction`. Модель отримує жорсткі правила (пре-промт): ставити лише по одному питанню, чекати на відповідь, та прощатися, коли користувач вирішує завершити інтерв'ю. 

```python
SYSTEM_INSTRUCTION = """
Ти - Senior Software Engineer та досвідчений технічний інтерв'юер у топовій IT-компанії.
Твоя мета: провести співбесіду з кандидатом на вказану посаду.
Правила:
1. Задавай по ОДНОМУ питанню за раз. Чекай на відповідь.
2. Питання мають бути технічними, практичними та відповідати рівню.
3. Anалізуй відповідь. Якщо неповна - задай уточнююче питання. Якщо повна - переходь до наступної теми.
4. Коли кандидат скаже "Завершити", попрощайся.
"""

```

Завершення бесіди можливе як текстовим "Завершити" (або іншими подібними фразами) так і кнопкою "Завершити бесіду".   
У випадку натискання кнопки нічого не відправляється в чат, а LLM просто готує Scorecard на основі опитування.  
Посада (наприклад, "Python Developer") динамічно підставляються у промпт із налаштувань інтерфейсу.  
Ріверь (наприклад "Middle") аналогічно змінюються із інтерфейсу.   
Зміна НЕ можлива під час бесіди, не впливає на опитування.

**Дебаг-режим:** Для тестування роботи застосунку вбудовано режим "Enable Debug Logging". Якщо він активований, до `system_instruction` додається вказівка проводити співбесіду у форматі простих текс-тестів (А, Б, В, Г) для швидкості тестування функціоналу LLM та доступності серверів.

## 3. Структурований вивід (FeedbackScorecard)

Генерація фінального Scorecard. Щоб уникнути галюцинацій та пере-форматування, використовується жорстка типізація через бібліотеку **Pydantic**. Схема `FeedbackScorecard` передається в параметр `response_schema`, що змушує Gemini повертати тільки валідний JSON.

```python
# Pydantic схема (schemas.py)
from pydantic import BaseModel, Field

class FeedbackScorecard(BaseModel):
    """Схема для фінального фідбеку після співбесіди."""
    technical_score: int = Field(description="Оцінка технічних знань від 1 до 10")
    communication_score: int = Field(description="Оцінка навичок комунікації від 1 до 10")
    # ...

# Ггенерація JSON (core.py)
def get_feedback(chat_history: str) -> dict:
    try:
        model = genai.GenerativeModel('gemini-3.1-flash-lite')
        prompt = f"Проаналізуй цю історію співбесіди та згенеруй фідбек-скоркард:\n\n{chat_history}"
        
        response = model.generate_content(
            prompt,
            generation_config=genai.types.GenerationConfig(
                response_mime_type="application/json",
                response_schema=FeedbackScorecard,
                temperature=0.1
            )
        )

```

## 4. Обробка помилок та лімітів

Робота з безкоштовним API (Free Tier) вимагає надійного перехоплення винятків ну і щоб модель відповідала взагалі.   
У проєкті реалізовано `try:...except` блоки для обробки критичних ситуацій:

```python
try:...except (core.py)
except exceptions.ResourceExhausted:
        raise PermissionError("Resource Exhausted: Перевищено ліміти Free Tier. Спробуйте пізніше.")

        raise RuntimeError(f"Помилка генерації фідбеку: {e}")

```

## 6. Встановлення та запуск (noDocker через uv)

### Крок 1: Клонування репозиторію
Відкрийте термінал (або PowerShell на Windows) у папці, куди хочете зберегти проєкт, і виконайте команду для завантаження коду з GitHub:
```bash
git clone [https://github.com/Bileneret/NLP.git](https://github.com/Bileneret/NLP.git)

```

### Крок 2:

Зайдіть всередину необхідної папки.

```bash
cd NLP/prac4/ai_interviewer

```
<sub>(ну або просто відкрийте термінал заново у цій папці)</sub>

### Крок 3: Налаштування Google API

1. Створіть у корені папки `ai_interviewer` звичайний текстовий файл і назвіть його строго `.env`
2. Перейдіть на сайт [Google AI Studio](https://aistudio.google.com), згенеруйте там безкоштовний ключ Free Tier.
3. Відкрийте створений файл `.env.txt` у будь-якому редакторі тексту (можна просто блокнот) та вставте туди ваш ключ у такому форматі: (замініть `AIzaS15y` на ваш ключ).  
   ```env
   GEMINI_API_KEY=AIzaS15y...

   ```
4. Приберіть розширення `.txt` зі створеного файлу. Підтвердіть зміни. Файл має називатись ТІЛЬКИ `.env` та мати тип: `Файл "ENV"`.

### Крок 4: Створення віртуального оточення

Ініціалізуйте ізольовану "бульбашку" для бібліотек, щоб вони не забруднювали системний Python:

```bash
python -m uv venv

```

### Крок 5: Встановлення всіх залежностей

Запустіть синхронізацію, яка автоматично прочитає конфіг `pyproject.toml` та встановить необхідні бібліотеки.

```bash
python -m uv sync
```

### Крок 6: Запуск веб-інтерфейсу

Запустіть застосунок через менеджер пакетів:

```bash
python -m uv run streamlit run src/interviewer/main.py
```

Після цього у вашому терміналі з'явиться локальне посилання (зазвичай `http://localhost:8501`), а в браузері автоматично відкриється вкладка із симулятором співбесіди. Дайте застосунку трохи подумати. Під час початку співбесіди аналогічно очікується невелика затримка, дайте LLM подумати. Дякую за увагу.


## 7. Альтернативний запуск (Docker)

Якщо ви не хочете встановлювати Python або `uv` на свій комп'ютер, проєкт можна запустити в повністю ізольованому Docker-контейнері. Для цього у вас має бути встановлений і запущений [Docker Desktop](https://www.docker.com/products/docker-desktop/). 

### Крок 1: Клонування та перехід у папку (якщо не зробили раніше)

```bash
git clone https://github.com/Bileneret/NLP.git
cd NLP/prac4/ai_interviewer
```

### Крок 2: Створення файлу `.env`

Створіть файл `.env` у папці `ai_interviewer` та пропишіть туди свій токен  
```env
GEMINI_API_KEY=token
```  
(детальніше дивіться у [Пункт №6 Крок №3](#крок-3-налаштування-google-api)) 

### Крок 3: Збирання Docker-образу

Запустіть команду збирання контейнера (крапка в кінці обов'язкова - вона вказує на поточну папку):

```bash
docker build -t ai-interviewer .
```

### Крок 4: Запуск контейнера

Запустіть зібраний образ, вказавши локальний порт `8501` та автоматично підтягніть API key з файлу `.env`:

```bash
docker run -p 8501:8501 --env-file .env ai-interviewer
```

Після цього відкривайте браузер і переходьте за адресою `http://localhost:8501` - застосунок готовий до роботи.
